# Инструменты 1С

Основное дерево команд в activity bar — **Инструменты 1С**. Появляется после открытия проекта 1С (каталог с `packagedef`).

![Дерево команд «Инструменты 1С»](../walkthrough/images/step3.png)

## Разделы дерева

- **Информационные базы** — создание, обновление, загрузка/выгрузка DT, инициализация данных.
- **Конфигурация** — загрузка и выгрузка `src/cf`, работа с `1Cv8.cf`, инкрементальная загрузка/выгрузка, сборка и разбор.
- **Расширения** — загрузка и выгрузка `src/cfe`, работа с `*.cfe`, сборка и разбор.
- **Внешние файлы** — сборка и разбор `epf`/`erf`, очистка кэша.
- **Поддержка** и **Поставка** — команды поддержки конфигурации и подготовки файлов поставки.
- **Зависимости** — инициализация `packagedef`, структуры проекта, Git, OneScript, OPM и зависимостей.
- **Служебные файлы** — создание `.gitignore`, `.gitattributes`, основного [профиля запуска](launch-profiles.md) и файлов `tools/*` ([service-files.md](service-files.md)).
- **Запуск**, **Тестирование**, **Установить версию**, **Конфигурации запуска**, **Навыки для AI**, **Помощь и поддержка**.

Панель состоит из двух плашек: **Команды** (дерево команд) и **Помощь и поддержка** (ссылки на документацию и поддержку).

Активный [профиль запуска](launch-profiles.md) выбирается в статусной строке (слева) и подставляется во все команды vrunner.

## Выбор расширений для команд

Команды группы **Расширения** (загрузка и выгрузка `src/cfe` и `*.cfe`, загрузка из `objlist.txt`, обновление в ИБ, сборка, разбор), а также **Установить версию → Расширения** работают не со всеми каталогами `src/cfe`, а с выбранным подмножеством. Это удобно, когда в проекте много расширений, а разработка идёт по нескольким: не нужно ждать, пока отработают все.

Источник выбора определяется так:

1. **Настройка `1c-platform-tools.extensions.selected`.** Если в `settings.json` задан список имён каталогов расширений — команды используют его **без окна выбора**. Имена сравниваются без учёта регистра; отсутствующие в `src/cfe` — пропускаются.
2. **Интерактивный выбор.** Если настройка пуста, при запуске команды показывается выпадающий список с чекбоксами по каталогам `src/cfe`:
   - изначально отмечены все;
   - снятый выбор запоминается для проекта (локально, не коммитится) и подставляется при следующем запуске любой команды расширений;
   - если оставить отмеченными все — выбор сбрасывается, и новые расширения подхватываются автоматически.

Пример `.vscode/settings.json` — команды будут работать только с этими двумя расширениями, без окна выбора:

```jsonc
{
  "1c-platform-tools.extensions.selected": [
    "РасширениеПоЗарплате",
    "РасширениеПоБюджетированию"
  ]
}
```

Пустой список (`[]`) или отсутствие настройки возвращает интерактивный выбор.

## Запуск команд: задачи VS Code

По умолчанию команды [vanessa-runner](https://github.com/vanessa-opensource/vanessa-runner) и `opm` (установка зависимостей, запуск oscript-задач) выполняются как **задачи VS Code** (Tasks). Это даёт:

- **«Rerun Last Task».** Последний запуск повторяется командой палитры **Tasks: Rerun Last Task** (по умолчанию без хоткея, можно назначить свой).
- **Запуск из списка задач и `tasks.json`.** Команды видны в **Tasks: Run Task**, их можно вынести в `tasks.json`, навесить хоткей или включить в составную задачу.
- **Единый статус и вывод.** Прогон идёт в панели задачи; вывод виден полностью, задачу можно остановить (она завершает всё дерево процессов `cmd → oscript → 1cv8`).

Отключить и вернуться к обычному интерактивному терминалу можно настройкой `1c-platform-tools.execution.useTasks: false`.

### Свои задачи в `tasks.json`

Команды vrunner можно описывать вручную в `.vscode/tasks.json` (тип `1c-vrunner`):

```jsonc
{
  "version": "2.0.0",
  "tasks": [
    {
      "type": "1c-vrunner",
      "command": "vanessa",
      "args": ["--settings", "env.json"],
      "label": "1C: Прогнать Vanessa",
      "group": "test"
    },
    {
      "type": "1c-vrunner",
      "command": "compile",
      "args": ["--src", "src/cf"],
      "label": "1C: Собрать конфигурацию"
    }
  ]
}
```

Запуск — **Tasks: Run Task** → выбрать задачу. Поле `command` — это команда vrunner (`vanessa`, `compile`, `syntax-check`, `init-dev` и т.д.), `args` — её аргументы.

## Установка OneScript

Команда **Установить OneScript** в разделе **Зависимости** ставит выбранную версию через [OVM](https://github.com/oscript-library/ovm). Сам `ovm.exe` расширение загружает из релизов GitHub и держит в своём кэше рядом с остальными внешними компонентами: он обновляется при выходе новой версии и переустанавливается командой **1C: Внешние компоненты: Обновить внешние компоненты**.

Переопределить загрузку можно в разделе настроек **Внешние компоненты**:

| Настройка               | Назначение                                            |
|-------------------------|-------------------------------------------------------|
| `components.ovmFile`    | путь к своему `ovm.exe` вместо загрузки с GitHub      |
| `components.ovmAutoload`| выключает загрузку, если путь выше не задан           |

## Избранное и палитра

- Часто используемые команды добавьте в избранное: **1C: Дерево: Настроить избранное** — раздел «Избранное» появится вверху дерева.
- Все команды доступны из палитры (`Ctrl+Shift+P` → введите «1C»).

## Скрытие групп

Ненужные группы дерева **Команды** можно скрыть: кнопка-глаз в шапке плашки (справа от «Избранного») открывает список групп с чекбоксами — снимите отметку, чтобы скрыть. Когда что-то скрыто, значок меняется на перечёркнутый глаз.

Выбор хранится на пользователя (не в `settings.json` проекта). «Избранное» не затрагивается: закреплённые команды остаются видны независимо от скрытых групп.

## Настройки

- `1c-platform-tools.paths.*` — пути к исходникам (`src/cf`, `src/cfe`, `src/epf`, `src/erf`) и результатам сборки.
- `1c-platform-tools.extensions.selected` — список расширений (имена каталогов в `src/cfe`), с которыми работают команды группы «Расширения» и установка версии расширений. Пусто — интерактивный выбор с запоминанием, см. [Выбор расширений для команд](#выбор-расширений-для-команд).
- `1c-platform-tools.vrunner.*` — путь и параметры vanessa-runner; по умолчанию используется `oscript_modules/bin/vrunner.bat` проекта, если он есть.
- `1c-platform-tools.execution.useTasks` — запускать команды как задачи VS Code (Rerun Last Task, запуск из списка задач); по умолчанию `true`. `false` — обычный интерактивный терминал.
- `1c-platform-tools.useIbcmd` — выполнять команды через `ibcmd` без GUI (нужен компонент сервера 1С).
- Выполнение в Docker — [docker.md](docker.md).
