# Служебные файлы

Служебные файлы — это вспомогательные файлы проекта 1С (настройки запуска, параметры тестов, исключения Git), которые расширение создаёт из готовых шаблонов. Файлы не перезатираются: если файл уже есть — он открывается, если нет — создаётся.

Раздел **Служебные файлы** доступен в дереве **Инструменты 1С** (свёрнут по умолчанию) и через палитру (`Ctrl+Shift+P` → «1C: Служебные файлы»).

## Что можно создать

| Подпись в дереве | Файл | Назначение | В базовом наборе |
| --- | --- | --- | --- |
| `.gitignore` | `.gitignore` | исключения для проекта 1С | да |
| `.gitattributes` | `.gitattributes` | text/binary для файлов 1С | да |
| **Осн. профиль запуска** | `env.json` / `autumn-properties.json` | базовый [профиль запуска](launch-profiles.md) (формат по версии vrunner) | да |
| **CI: прогон тестов** | `tools/vrunner.json` | настройки прогона тестов в CI (BDD, дымовые) | нет |
| **CI: инициализация данных** | `tools/vrunner.init.json` | настройки шага инициализации данных в CI | нет |
| **VA: параметры тестов** | `tools/VAParams.json` | параметры Vanessa Automation для прогона тестов | нет |
| **VA: параметры инициализации** | `tools/VAParams.init.json` | параметры Vanessa Automation для инициализации данных | нет |
| **xUnit: параметры** | `tools/xUnitParams.json` | параметры дымовых тестов xUnit | нет |
| **YaXunit: параметры** | `tools/yaxunit.json` | конфигурация YAxUnit | нет |
| **Syntax: исключения** | `tools/syntax-check-excludes.txt` | исключения синтаксического контроля | нет |
| **1cpt: hooks** | `.1cpt/hooks.json` | [хуки](hooks.md) `pre`/`post`/`onError` на команды расширения | нет |

Подпись «Осн. профиль запуска» отличает базовый файл настроек от именованных профилей; фактическое имя файла зависит от установленного vanessa-runner (`env.json` для 2.x, `autumn-properties.json` для 3.x).

## Создание

- **Создать служебные файлы** (`$(new-file)`) — QuickPick со всеми файлами; наличие отмечено `$(check)`, отсутствие — `$(add)`. Здесь же пункт **Базовый набор**.
- **Базовый набор** — создаёт только отсутствующие `.gitignore`, `.gitattributes` и основной профиль запуска одним действием (без диалогов).
- Отдельные пункты дерева создают/открывают конкретный файл.

## Основной профиль запуска — особый случай

Основной профиль — одновременно служебный файл и базовый [профиль запуска](launch-profiles.md). При его создании (кроме пакетного «Базового набора») расширение предлагает отметить опциональные **секции команд** — `vanessa`, `xunit`, `syntax-check` (базовые настройки подключения добавляются всегда). Секции записываются в формате установленного vrunner:

- 2.x (`env.json`) — плоские секции `vanessa` / `xunit` / `syntax-check`;
- 3.x (`autumn-properties.json`) — каскад `vrunner.test.vanessa` / `vrunner.test.xunit` / `vrunner.validate.syntax-check`.

Выбранные секции ссылаются на файлы из `tools/` (`VAParams.json`, `xUnitParams.json` и т.п.), поэтому эти файлы стоит создавать вместе с соответствующей секцией.

## CI-файлы

`tools/vrunner.json` и `tools/vrunner.init.json` — файлы настроек для CI (совместимы с [jenkins-lib](https://github.com/firstBitMarksistskaya/jenkins-lib)): один для прогона тестов, другой для шага инициализации данных. На vrunner 3 они создаются в том же формате `autumn`, что и основной профиль (та же база `vrunner` + секция `test.vanessa`), отличаясь только файлом параметров VA.
