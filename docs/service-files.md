# Служебные файлы

Служебные файлы — это вспомогательные файлы проекта 1С (настройки запуска, параметры тестов, исключения Git), которые расширение создаёт из готовых шаблонов. Файлы не перезатираются: если файл уже есть — он открывается, если нет — создаётся из шаблона.

Раздел **Служебные файлы** доступен в дереве **Инструменты 1С** (свёрнут по умолчанию) и через палитру (`Ctrl+Shift+P` → «1C: Служебные файлы»).

## Что можно создать

| Файл | Назначение | В базовом наборе |
| --- | --- | --- |
| `.gitignore` | исключения для проекта 1С | да |
| `.gitattributes` | text/binary для файлов 1С | да |
| `env.json` | базовый [профиль запуска](launch-profiles.md) | да |
| `tools/vrunner.json` | настройки vanessa-runner (пакетный прогон) | нет |
| `tools/vrunner.init.json` | настройки vanessa-runner (интерактивное открытие VA) | нет |
| `tools/VAParams.json` | параметры Vanessa Automation (пакетный прогон) | нет |
| `tools/VAParams.init.json` | параметры Vanessa Automation (интерактивно) | нет |
| `tools/xUnitParams.json` | параметры дымовых тестов xUnit | нет |
| `tools/yaxunit.json` | конфигурация YAxUnit | нет |
| `tools/syntax-check-excludes.txt` | исключения синтаксического контроля | нет |

## Создание

- **Создать служебные файлы** (`$(new-file)`) — QuickPick со всеми файлами; наличие отмечено `$(check)`, отсутствие — `$(add)`. Здесь же есть пункт **Базовый набор**.
- **Базовый набор** — создаёт только отсутствующие `.gitignore`, `.gitattributes` и `env.json` одним действием.
- Отдельные пункты дерева (**.gitignore**, **.gitattributes**, **env.json**) создают/открывают конкретный файл.

Файлы из каталога `tools/` создаются через пункт **Создать служебные файлы** — там они показаны полным списком.

## env.json — особый случай

`env.json` — одновременно служебный файл и базовый профиль запуска. При его создании (как и при создании именованного профиля) расширение предлагает отметить опциональные **секции команд** — `vanessa`, `xunit`, `syntax-check` (секция `default` добавляется всегда). Выбранные секции ссылаются на файлы из `tools/` (`VAParams.json`, `xUnitParams.json` и т.п.), поэтому эти файлы и стоит создавать вместе с соответствующей секцией.

После создания `env.json` становится активным профилем «По умолчанию». Подробнее о профилях, временных параметрах и статус-баре — в [Профилях запуска](launch-profiles.md).
