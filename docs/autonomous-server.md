# Автономный сервер

Автономный сервер (`ibsrv`) — это лёгкий сервер 1С:Предприятия, который публикует файловую ИБ проекта по HTTP без отдельной серверной установки. Расширение поднимает его для локальной разработки и отладки REST (HTTP-сервисы), SOAP (Web-сервисы) и OData: запустил — и сервисы доступны в браузере и из внешних клиентов.

Параметры сервера задаются в настройках расширения (`1c-platform-tools.server.*`), а не в `env.json` — последний регламентирован vanessa-runner. Путь к файловой ИБ берётся из строки подключения `--ibconnection` активного [профиля запуска](launch-profiles.md) (поддерживается только файловая ИБ `/F`).

## Статус-бар и меню

После открытия проекта 1С в статусной строке появляется элемент состояния сервера:

- `$(server) Сервер 1С` — остановлен;
- `$(sync~spin) Сервер 1С…` — запускается;
- `$(broadcast) Сервер 1С :<порт>` — запущен;
- `$(error) Сервер 1С` — ошибка (откройте журнал).

Клик открывает меню действий: **Запустить / Остановить / Перезапустить**, **Открыть в браузере**, **Выбрать публикуемые сервисы**, **Отладка через сервер**, **Показать журнал**, **Открыть конфиг публикации**. Те же действия доступны командами палитры в категории **1C: Автономный сервер**.

## Запуск

При запуске расширение:

1. находит бинарь `ibsrv` по каталогу установки платформы и версии (см. настройки);
2. определяет каталог файловой ИБ из `--ibconnection` активного профиля;
3. создаёт конфиг публикации `build/ibsrv/publication.yaml`, **если его ещё нет** (существующий не перезаписывается);
4. поднимает процесс `ibsrv` и ждёт готовности HTTP-эндпоинта;
5. показывает в статус-баре порт (зелёная иконка — сервер запущен), а в журнале — корневой адрес публикации.

Журнал процесса пишется в канал **Output → «1C: Автономный сервер»**.

### Конфиг публикации и ручные правки

Конфиг лежит в `build/ibsrv/publication.yaml` (каталог `build/` обычно в `.gitignore` — у каждого свой). Это **источник истины** для уже созданного файла:

- файл создаётся при первом запуске (или командой **Открыть конфиг публикации**) и больше не перезаписывается автоматически;
- можно остановить сервер, изменить параметры прямо в файле (например `server.port`) и запустить заново — новые параметры применятся (адрес и порт в статус-баре/браузере читаются из файла);
- при изменении набора публикуемых сервисов (см. ниже) файл **перегенерируется**, но серверные параметры (порт/хост/база/лицензии) сохраняются из текущего файла;
- каталог можно переопределить настройкой `server.dataPath`.

## Открыть в браузере

Действие **Открыть в браузере** предлагает корень публикации и OData `$metadata`. Адреса строятся по настройкам (`host`, `port`, `httpBase`), например `http://localhost:8314/ib/` и `http://localhost:8314/ib/odata/standard.odata/$metadata`. HTTP-сервисы доступны по `…/ib/hs/<корневой_URL>`, Web-сервисы — по `…/ib/ws/<имя>`.

## Выбор публикуемых сервисов

**Выбрать публикуемые сервисы** открывает множественный выбор по дереву метаданных (md-sparrow): OData, отдельные Web- и HTTP-сервисы.

- Отмеченные **«Все Web-сервисы» / «Все HTTP-сервисы»** — публикуется вся категория (по умолчанию).
- Снимите «Все …» и отметьте конкретные сервисы — публикуются только они. Корневой URL (HTTP) и синоним (Web) сервиса `ibsrv` берёт из метаданных, поэтому достаточно отметить имя.

Выбор хранится локально в рабочей области (`workspaceState`) и **не коммитится**. Если сервер запущен, расширение предложит перезапустить его, чтобы применить новый набор публикации. Если метаданные недоступны, в выборе остаются только категории.

## Отладка через сервер

**Отладка через сервер** включает порт отладки (`server.debug`, если выключен), поднимает/перезапускает сервер с отладкой и подключает отладчик расширения к порту отладки `ibsrv` (`server.debugPort`). Благодаря выдаче лицензий автономным сервером отлаживать серверный код HTTP/Web-сервисов можно без отдельной лицензии. Подробнее об отладчике — в [руководстве по отладке](debug.md).

## Совместимость с vanessa-runner 3

Сервер запускается на стабильном порту из настроек, поэтому к нему можно подключать прогоны vanessa-runner 3 через `--ibsrv-attach --ibsrv-port <порт>` — один сервер и для ручной работы с сервисами, и для тестов.

## Настройки

| Настройка | По умолчанию | Назначение |
| --- | --- | --- |
| `1c-platform-tools.server.platformPath` | пусто | Каталог установки платформы для поиска `ibsrv`. Пусто — автоопределение (`%PROGRAMFILES%\1cv8` в Windows, `/opt/1cv8/x86_64` и `/opt/1C/v8.3/x86_64` в Linux). `ibsrv` ищется в `bin/`, напрямую в каталоге версии и в самой базе — каталог `bin` в Linux-раскладке не обязателен. |
| `1c-platform-tools.server.platformVersion` | пусто | Версия платформы (`8.3.27.1936` или префикс `8.3.27`). Пусто — наибольшая установленная. |
| `1c-platform-tools.server.host` | `localhost` | Сетевой интерфейс (`localhost` / `any` / IP). |
| `1c-platform-tools.server.port` | `8314` | HTTP-порт. |
| `1c-platform-tools.server.httpBase` | `ib` | Базовый путь публикации (`ib` → `…/ib/`; пусто или `/` — корень сайта). |
| `1c-platform-tools.server.dataPath` | пусто | Каталог данных сервера и `publication.yaml`; относительный путь — от корня проекта. Пусто — `build/ibsrv`. |
| `1c-platform-tools.server.distributeLicenses` | `true` | Выдача клиентских лицензий (работа и отладка без отдельной лицензии). |
| `1c-platform-tools.server.publishOData` | `true` | Публиковать OData по умолчанию. |
| `1c-platform-tools.server.publishWebServices` | `true` | Публиковать Web-сервисы по умолчанию. |
| `1c-platform-tools.server.publishHttpServices` | `true` | Публиковать HTTP-сервисы по умолчанию. |
| `1c-platform-tools.server.debug` | `false` | Включить порт отладки сервера. |
| `1c-platform-tools.server.debugPort` | `1550` | Порт сервера отладки. |

Настройки `publish*` задают поведение по умолчанию; точечный выбор сервисов (см. выше) его переопределяет.
