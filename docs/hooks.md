# Хуки команд

Хуки позволяют выполнять произвольные команды оболочки **до** и **после** команд 1C: Platform Tools. Типовой кейс: перед заливкой исходников в ИБ автоматически поправить код, а после (или при ошибке) — вернуть исходники как было.

Хуки настраиваются в файле `1cpt/hooks.json` в корне рабочей области — один файл на проект, без привязки к [профилю запуска](launch-profiles.md).

## Создание файла

`hooks.json` входит в [служебные файлы](service-files.md), поэтому отдельно искать путь и заводить файл руками не нужно. Создать его можно тремя способами:

- **Дерево «Инструменты 1С»** → раздел **Служебные файлы** → пункт **hooks.json**. Клик создаёт файл из заготовки (если его ещё нет) и открывает его.
- **Палитра**: `Ctrl+Shift+P` → «1C: Служебные файлы: Создать служебные файлы…» → выбрать **hooks.json** в списке.
- Вручную — создать `1cpt/hooks.json` в корне проекта.

В **базовый набор** служебных файлов `hooks.json` не входит (хуки нужны не каждому проекту) — он создаётся только по явному выбору.

При создании файл заполняется заготовкой: проставлены `$schema` и `version`, таблица `hooks` пустая — остаётся добавить нужные команды.

## Формат файла

```jsonc
{
  "$schema": "https://raw.githubusercontent.com/yellow-hammer/vscode-1c-platform-tools/main/resources/schemas/hooks.schema.json",
  "version": 1,
  "hooks": {
    "1c-platform-tools.configuration.loadFromSrc": {
      "pre":     "node ./tools/rename-interface.js --to evERP",
      "post":    "git checkout -- src/cf",
      "onError": "git checkout -- src/cf"
    },
    "*": {
      "pre": "echo before any command"
    }
  }
}
```

Ключ хука — **VS Code command id** вида `1c-platform-tools.<домен>.<действие>` либо `"*"` (срабатывает на все команды). Привязка идёт к command id, а не к логическому имени операции: command id стабилен, а имена операций со временем меняются.

> Подсказки и валидация работают в редакторе благодаря JSON-схеме (`$schema` проставляется автоматически при создании файла). Схема знает точный список команд расширения: редактор автодополняет command id и подсвечивает опечатки и устаревшие id. Список генерируется из кода при сборке, поэтому остаётся актуальным при добавлении и переименовании команд.

## Фазы

| Фаза | Когда выполняется |
| --- | --- |
| `pre` | до команды; ненулевой код **прерывает** команду и запускает `onError` |
| `post` | после успешной команды (код 0) |
| `onError` | при неуспехе: ненулевой `pre`, ненулевой код команды или исключение |

Кейс «вернуть как было» закрывается тем, что один и тот же скрипт указывается и в `post`, и в `onError`.

## Значение фазы

Фаза задаётся одним из трёх способов:

- **строка** — одна команда оболочки;
- **массив** — последовательность команд;
- **объект** — `{ "command": string, "continueOnError"?: boolean, "timeout"?: number }`.

```jsonc
{
  "hooks": {
    "1c-platform-tools.configuration.loadFromSrc": {
      "pre": [
        "node ./tools/rename-interface.js --to evERP",
        { "command": "npm run lint", "continueOnError": true, "timeout": 60 }
      ]
    }
  }
}
```

- `continueOnError: true` — ненулевой код шага не прерывает выполнение (для `pre` — не отменяет команду).
- `timeout` — таймаут шага в секундах (по умолчанию 30).

## Отслеживание завершения

Для команд, запускаемых в терминале (без ожидания результата), `post` и `onError` выполняются после **фактического** завершения задачи. Это требует запуска через Task API: если для команды заданы `post` или `onError`, расширение использует задачу VS Code независимо от настройки `execution.useTasks`. Команды с одним лишь `pre` запускаются обычным путём.
