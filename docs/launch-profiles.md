# Профили запуска

Профиль запуска — это файл настроек vanessa-runner в корне проекта, который подставляется во все команды расширения через `--settings`. Так одни и те же команды (создание ИБ, загрузка конфигурации, запуск, тесты) выполняются над разными базами — локальной, dev, демонстрационной — без правки самих команд.

## Файлы профилей

- `env.json` — базовый профиль «По умолчанию».
- `env.<id>.json` — именованный профиль, где `<id>` становится его подписью (`env.dev.json` → профиль **dev**, `env.prod.json` → **prod**).

Профили обнаруживаются автоматически по именам файлов в корне проекта. `env.json` всегда идёт первым, остальные — по алфавиту. Сам файл `env.json` — это ещё и [служебный файл](service-files.md): его можно создать из дерева «Служебные файлы» или при первом обращении к выбору профиля.

## Статус-бар и выбор профиля

После открытия проекта 1С слева в статусной строке появляется элемент `$(variable-group) <профиль>` — текущий активный профиль. Клик открывает меню **Профиль запуска 1С**:

- **Не выбран** / список профилей — выбор активного профиля (отмечен `$(check)`). Состояние «Не выбран» подсвечивается жёлтым: команды vrunner пойдут без `--settings`, на дефолтных значениях.
- **Создать профиль…** — спрашивает идентификатор, создаёт `env.<id>.json` (с выбором секций команд, см. ниже) и делает его активным.
- **Открыть файл активного профиля** — открывает текущий `env.*.json` в редакторе.
- **Временные параметры** / **Сбросить параметры** — см. ниже.

Активный профиль хранится локально в рабочей области (`workspaceState`) и **не коммитится** — у каждого участника команды может быть свой выбор. Если файл активного профиля удалить, выбор автоматически сбрасывается на «Не выбран».

## Временные параметры

Поверх активного профиля можно задать временные переопределения отдельными флагами — они применяются ко всем командам vrunner, не меняя файл профиля. Доступны из меню профиля (**Временные параметры**) или командой палитры **1C: Конфигурации запуска: Задать временные параметры запуска**:

| Параметр | Флаг vrunner |
| --- | --- |
| Строка подключения к ИБ | `--ibconnection` (файловая `/F<путь>`, серверная `/S<сервер>\<имя_ИБ>`) |
| Пользователь ИБ | `--db-user` |
| Пароль ИБ | `--db-pwd` |
| Версия платформы | `--v8version` |
| Дополнительные параметры | `--additional` |

Заданы только непустые поля — остальное берётся из файла профиля. Когда переопределения активны, в статус-баре рядом с профилем появляется `$(pencil)`. **Сбросить параметры** очищает их. Временные параметры, как и выбор профиля, хранятся локально и не коммитятся.

## Секции команд в env.json

При создании `env.json` или именованного профиля расширение предлагает отметить опциональные секции (секция `default` добавляется всегда):

- **vanessa** — сценарные тесты Vanessa Automation (BDD/feature).
- **xunit** — дымовые тесты xUnit.
- **syntax-check** — синтаксический контроль конфигурации.

Секции добавляют в файл готовые блоки настроек, которые ссылаются на файлы из каталога `tools/` (`VAParams.json`, `xUnitParams.json` и т.п.). Сами эти файлы создаются как [служебные](service-files.md).

## Настройки

- `1c-platform-tools.defaultEnvProfile` — идентификатор профиля по умолчанию (имя `X` из `env.X.json`; пусто — базовый `env.json`). В отличие от локального выбора, это значение можно закоммитить в настройки репозитория как общекомандный дефолт; локальный выбор его переопределяет.

## Поддержка vanessa-runner 3.x

Расширение работает и с vanessa-runner 2.x (LTS), и с 3.x — версия определяется автоматически (`vrunner version` / `vrunner --version`). При установленном vrunner 3 это прозрачно для вас:

- команды приводятся к новому синтаксису (`vrunner vanessa` → `vrunner test vanessa`, `compile` → `cf compile` и т.д.), с переносом опций перед позиционными аргументами;
- активный `env.json` на лету конвертируется в формат настроек 3.0 (`autumn-properties`: пространство имён `vrunner.<команда>`, ключи без `--`) и подставляется в `--settings` — сам файл `env.json` править не нужно.

Формат `env.json` в проекте остаётся прежним (2.x); конвертация выполняется только для запуска под vrunner 3.

## Связанные руководства

- [Служебные файлы](service-files.md) — создание `env.json` и файлов `tools/*`, на которые ссылаются секции команд.
- [Инструменты 1С](tools.md) — команды vanessa-runner, к которым применяется активный профиль.
