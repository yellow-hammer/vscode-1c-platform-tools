# Профили запуска

Профиль запуска — это файл настроек vanessa-runner в корне проекта, который читается раннером автоматически и определяет, над какой базой и с какими параметрами выполняются команды расширения (создание ИБ, загрузка конфигурации, запуск, тесты, публикация сервера, отладка). Так одни и те же команды работают над разными базами — локальной, dev, демонстрационной — без правки самих команд.

## Формат зависит от версии vanessa-runner

Расширение само определяет установленную версию vanessa-runner и выбирает формат файла настроек:

| Версия vrunner | Базовый профиль | Именованный профиль |
| --- | --- | --- |
| 2.x | `env.json` | `env.<id>.json` (`env.dev.json` → профиль **dev**) |
| 3.x | `autumn-properties.json` | `autumn-properties.<id>.json` |

vanessa-runner 3 — это отдельный инструмент со своим форматом настроек (каскад `vrunner.<команда>.<опция>`), поэтому расширение **не конвертирует** `env.json` на лету. Профили обнаруживаются автоматически по именам файлов в корне проекта: базовый идёт первым, остальные — по алфавиту.

Перенос существующих настроек 2.x в формат 3.0 выполняется официальным инструментом из состава vanessa-runner:

```
oscript oscript_modules/vanessa-runner/tools/migrate26to30.os --input vrunner.json --output autumn-properties.json
```

## Панель «Профиль запуска»

Под панелью «Команды» находится плашка **Профиль запуска** — что именно видят команды vrunner:

- **Профиль** — активный профиль (клик открывает выбор).
- **vanessa-runner: `<версия>`** — определённая версия и синтаксис (2.x/3.x); клик — определить версию заново (после переустановки раннера).
- **Файл настроек** — базовый файл схемы; если его нет, пункт подсвечен предупреждением, команды заблокированы, клик создаёт файл.
- **ИБ** — строка подключения из файла настроек.
- **Временные параметры** — разовые переопределения (см. ниже).

## Статус-бар и выбор профиля

В статусной строке слева — элемент `$(rocket) <профиль>`. Если базового файла настроек нет, элемент подсвечен предупреждением (команды заблокированы). Клик открывает меню **Профиль запуска 1С**:

- Список профилей — выбор активного (отмечен `$(check)`).
- **Создать профиль…** — спрашивает идентификатор, создаёт `env.<id>.json` / `autumn-properties.<id>.json` и делает его активным.
- **Открыть редактор профиля** — [форма-редактор](service-files.md) файла настроек.
- **Временные параметры** / **Сбросить параметры** — см. ниже.
- **Определить версию заново** — перечитать версию vanessa-runner.

Как раннер читает файл настроек из корня проекта сам, базовый профиль не требует `--settings` — расширение добавляет `--settings <файл>` только для **именованного** профиля. Активный профиль хранится локально в рабочей области (`workspaceState`) и **не коммитится**.

## Временные параметры

Поверх активного профиля можно задать разовые переопределения отдельными флагами — они применяются ко всем командам vrunner, не меняя файл профиля:

| Параметр | Флаг vrunner |
| --- | --- |
| Строка подключения к ИБ | `--ibconnection` (файловая `/F<путь>`, серверная `/S<сервер>\<имя_ИБ>`) |
| Пользователь ИБ | `--db-user` |
| Пароль ИБ | `--db-pwd` |
| Версия платформы | `--v8version` |
| Дополнительные параметры | `--additional` |

Заданы только непустые поля — остальное берётся из файла профиля. Когда переопределения активны, в статус-баре рядом с профилем появляется `*`. Временные параметры, как и выбор профиля, хранятся локально и не коммитятся.

## Секции команд при создании профиля

При создании базового профиля расширение предлагает отметить опциональные секции (базовые настройки подключения добавляются всегда):

- **vanessa** — сценарные тесты Vanessa Automation (BDD/feature);
- **xunit** — дымовые тесты xUnit;
- **syntax-check** — синтаксический контроль конфигурации.

Секции добавляют в файл готовые блоки настроек в формате установленного vrunner (плоские секции в `env.json`, каскад `vrunner.test.*` / `vrunner.validate.*` в `autumn-properties.json`), которые ссылаются на файлы из каталога `tools/`.

## Редактор профиля

Файл настроек можно править как JSON вручную или через форму-редактор: пункт **Открыть редактор профиля** в меню профиля либо клик по строке «Файл настроек» на панели «Профиль запуска» открывает `env*.json` / `autumn-properties*.json` в удобной форме.

- Опции сгруппированы по смыслу (Подключение, Платформа, ibcmd и СУБД, ibsrv, EDT, Хранилище, Кластер, Vanessa, xUnit, синтаксический контроль, Запуск); нестандартные и редкие — в свёрнутой группе «Прочее».
- Каталог опций и их описания собраны из официальной схемы vanessa-runner 2.x и исходников 3.x, дополнены кураторским слоем (уточнения типов, подписи значений enum).
- **Добавить параметр** и **Команда…** — выбор из известных опций через QuickPick с поиском по описанию; для 3.x параметры можно задавать пер-командно (каскад `vrunner.<команда>`).
- Правки точечные: форматирование и комментарии JSON сохраняются (редактор поверх того же файла — переключение между формой и текстом не теряет изменений).
- Каталоги опций коммитятся в репозиторий (редактор работает офлайн); обновить их можно задачей VS Code **«Обновить каталоги опций vrunner»** (генератор скачивает схему 2.x и исходники нужного тега 3.x).

## Где применяется активный профиль

Профиль — единая точка настройки: его учитывают **все** пути расширения.

- **Команды vanessa-runner** — `--settings` активного профиля подставляется централизованно; временные параметры и версия платформы берутся из профиля.
- **Панель тестирования** — vanessa/xunit/yaxunit и синтаксический контроль запускаются под активным профилем, пути отчётов читаются из его файла настроек (с учётом схемы).
- **Автономный сервер (ibsrv)** — публикует ИБ активного профиля и его версию платформы; при смене профиля предлагает перезапуск, если работает на другой базе.
- **Отладка 1С** — строка подключения, учётные данные и версия платформы берутся из активного профиля.

## Настройки

- `1c-platform-tools.defaultEnvProfile` — идентификатор профиля по умолчанию (имя `X` из `env.X.json` / `autumn-properties.X.json`; пусто — базовый). В отличие от локального выбора, это значение можно закоммитить как общекомандный дефолт; локальный выбор его переопределяет.
