# Запуск 1С: Конфигуратор и Предприятие

Запускай Конфигуратор или Предприятие через **MCP** (сервер 1C: Platform Tools MCP) или команды расширения. Выполняй вызов сам, не проси пользователя открывать палитру.

## Когда вызывать

| Запрос пользователя (примеры)                | Действие                              |
|----------------------------------------------|---------------------------------------|
| Запусти Конфигуратор, открой конфигуратор    | `run_designer` / команда designer     |
| Запусти Предприятие, открой 1С, запусти базу | `run_enterprise` / команда enterprise |

## Через MCP (предпочтительно)

| Задача                 | MCP tool         |
|------------------------|------------------|
| Запустить Конфигуратор | `run_designer`   |
| Запустить Предприятие  | `run_enterprise` |

### Параметр projectPath

Обязательный. Абсолютный путь к корню проекта 1С (каталог с `packagedef` и `env.json`).

- Если пользователь указал путь к проекту — используй его.
- Если не указал — возьми корень текущего workspace (первая папка). Для проекта 1С это обычно каталог с `packagedef`.

### Важно

Не используй для запуска Конфигуратора/Предприятия инструменты `launch_edit` и `launch_run` — они для других задач (редактирование списка задач, запуск задачи по метке).

## Через Execute Command (если MCP недоступен)

| Задача                 | Command ID                         |
|------------------------|------------------------------------|
| Запустить Предприятие  | `1c-platform-tools.run.enterprise` |
| Запустить Конфигуратор | `1c-platform-tools.run.designer`   |

## Примеры

- **MCP:** вызови инструмент `run_designer` с `projectPath` = `C:\Projects\My1C` (или путь корня workspace).
- **MCP:** вызови `run_enterprise` с тем же `projectPath`.
- **Команда:** выполни `1c-platform-tools.run.designer` (projectPath берётся из текущего открытого проекта в расширении).

Настройки подключения к ИБ и платформе заданы в `env.json` проекта; не запускай 1cv8 вручную.
